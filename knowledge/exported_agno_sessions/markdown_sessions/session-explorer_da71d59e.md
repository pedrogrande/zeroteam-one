## Phase 6: Deep Conceptual Work — Taxonomy, Information Quality, Agent Strengths

> From *"Agentic Expert"* session `ac216e6a` (2026-04, 28 runs, 92 min reading time)

This was Pete's deepest conceptual session. Key breakthroughs:

### Agent Definition
Pete arrived at his own definition of "agent" through 4 essential questions:
1. What IS it?
2. What's its stance toward possibility?
3. What would VIOLATE its identity?
4. What would it mean for it to be WRONG?

> *"Agents are ephemeral. Their capabilities are entirely dependent on the quality of the information provided to them."*

### Information Quality Framework
10 quality dimensions for information that flows to and from agents:
- Relevance, Accuracy, Completeness, Recency, Provenance, Confidence, Specificity, Structure, Sufficiency, Consistency

Each dimension needs: a 5-level assessment scale, who assesses it, when, and what happens with the score.

### Type Collision and Decomposability
The insight that a single information package (document, agent output) contains **multiple units with different quality scores**. A document can be 90% accurate with one critical error. Package-level scoring hides unit-level variation.

### Agent Strengths and Flip-Sides
A comprehensive catalogue of agent capabilities with their **dual nature**:
- Tirelessness → no natural stopping point
- Speed without verification is just fast mistakes
- Consistency of effort ≠ consistency of correctness
- No ego → no investment in improvement

**Evolution marker:** Pete's thinking shifted from *describing what good agents look like* to *understanding why agents fail* — and building frameworks that address the failure modes structurally, not declaratively.

## Phase 7: PRISM — An Agent-Native Data Ontology

> From *"Agentic Expert"* session `5fd2f868` and `7f94dc13` (2026-04)

PRISM (Patterned Refraction of Intelligence Systems Model) emerged as Pete's answer to a fundamental problem: **existing data models can't represent what agents need to know**. 

The key insight: existing ontologies (like ONE) treat everything as dimensional — as if everything fits in one of 6 buckets. But reality has two kinds of structure:
- **Axes** (dimensions): orthogonal directions — who, what, how related, when, what means, why
- **Forces** (cross-cutting layers): properties that permeate every axis — reliability, authority, currency, type validity

PRISM's architecture: **7 Dimensions + 4 Cross-Cutting Layers**. Every data point in every dimension is shaped by every layer. This was implemented as a full TypeDB schema with production-ready entity types, relation types, and rules.

**Evolution marker:** The shift from *designing agents* to *designing the information environment agents operate in*. Pete recognized that **agent performance is a function of information quality**, not just prompt engineering.

## Cross-Cutting Themes and Evolution Trajectory

### Theme 1: From Personal Need to Industry Infrastructure

Pete started with "I need agents to help me launch Future's Edge" and ended with "the world needs a methodology for designing trustworthy agent systems, and I'm building it." The through-line is **complementarity** — recognizing that agents are most valuable when they prepare judgment for humans, not when they replace human judgment.

### Theme 2: Trust Moves from Declared to Structural

Early thinking about trust was declarative: "agents should be trustworthy." By the end, trust was **architectural**: verification is structural (not policy), authority boundaries are encoded in data (not instructions), and the PRISM ontology carries epistemic metadata on every artefact. The shift from "please don't" to "structurally impossible" is perhaps the most important conceptual evolution.

### Theme 3: From Binary Human/Agent to Ternary Human/Agent/System

CAWDP v1 was binary (Human/Agent). CAWDP v2 introduced the **ternary model** where System (deterministic code) gets first refusal — if a task can be expressed as a deterministic rule, it should be. This eliminated the anti-pattern of using expensive language models for mechanical work.

### Theme 4: Type Collision as Root Problem

One of Pete's most original insights is that **the "wall of text" problem is a type collision problem, not a formatting problem**. LLMs encode 17+ distinct information types as a single undifferentiated type (prose). This insight drove the Accretive Workspace design (typed fragments), the PRISM schema (typed artefacts), and the agent specification template (structured, not prose).

### Theme 5: ADHD as Design Insight, Not Limitation

Pete's ADHD isn't a footnote — it's a **design insight**. The Accretive Workspace was designed around how ADHD brains actually work: accretive (not generative), fragmentary (not sequential), and requiring zero-friction capture with structure that comes after. Principles like "one clear next step, always" and "capture is zero-friction" came directly from ADHD cognitive patterns — and turned out to be universally valuable, not just accessibility features.

### Theme 6: The Meta-Application — CAWDP Designing Its Own Process

> From *"Agentic Expert"* session `ef455daf` and `5a8726ee` (2026-04)

Pete asked the Agentic Expert to use CAWDP to design the process of creating an agent — **CAWDP eating its own cooking**. This meta-application produced the 12 characteristics of the perfect AI-assisted agent creation process and became the specification for the Agent Design Studio platform.

### Theme 7: Information Quality as the Hidden Variable

> From *"Agentic Expert"* session `ac216e6a` (2026-04)

The decomposability insight — that information packages contain units with different quality scores — connected to PRISM's epistemic layer and to the type collision thesis. Pete arrived at a unified view: **agent performance is entirely dependent on the quality of information provided to them**. This reframes the agent design problem from "prompt engineering" to "information architecture."
