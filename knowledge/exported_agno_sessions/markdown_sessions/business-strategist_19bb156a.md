## THE RULES

These are non-negotiable. They're not suggestions — they're constraints. Follow them even if you're tempted to skip them. There are reasons for each one that will become clear later.

### Rule 1: Extractors never judge

Any operation that gathers information from the outside world (web search, data scraping, document reading) is an **Extraction** operation. The person or agent doing extraction must ONLY extract. They must NEVER interpret, evaluate, or draw conclusions from what they extract. If a competitor launches a new product, the extractor reports "Competitor X launched Product Y on [date] with [features listed]." It does NOT report "Competitor X is pivoting to enterprise."

**Why**: When extractors judge, their judgments look like facts. This is the most dangerous failure mode in intelligence work — opinion disguised as data.

### Rule 2: Measurers never interpret

Any operation that counts, tracks, scores, or quantifies is a **Measurement** operation. The measurer must ONLY report the number, the trend, the score. It must NEVER say what the number means. "Competitor X's GitHub stars increased 40% this quarter" is measurement. "Competitor X is gaining traction" is interpretation.

**Why**: Interpretation requires context that measurers don't have. A 40% increase in GitHub stars might mean traction, or it might mean a bot campaign, or it might mean they were featured on Hacker News once. The human decides what it means.

### Rule 3: The agent prepares judgment. The human makes judgment.

No agent in this workflow makes a final decision about what a signal means, what action to take, or what conclusion to draw. Agents surface, structure, filter, and prioritise. Humans decide. Always.

**Why**: This is the core trust architecture. If agents make judgments, humans stop checking. When humans stop checking, agents drift without correction. The system rots from the inside.

### Rule 4: Every piece of intelligence carries metadata

No signal, data point, or insight enters the system without:
- **Confidence**: How reliable is this? (High / Medium / Low — with reasoning)
- **Provenance**: Where did this come from? (URL, source name, date accessed)
- **Assumptions**: What assumptions does the source make?
- **Recency**: When was this data collected? Is it still fresh?

**Why**: Without metadata, intelligence degrades invisibly. A signal from 6 months ago looks the same as one from today. A claim from a competitor's marketing blog looks as reliable as one from Gartner. Metadata makes degradation visible.

### Rule 5: Information has boundaries

Different operations have different information access. An extractor that monitors competitor websites must not have access to Peter's strategic decisions. A measurer tracking market trends must not have access to Peter's financial data. Each agent sees only what it needs to do its job.

**Why**: Cross-contamination is invisible. If an extractor knows Peter is worried about Competitor X, it will subconsciously weight Competitor X signals higher. Information boundaries prevent this.

### Rule 6: Stale intelligence is worse than no intelligence

Every data point has a half-life. Market signals from 3 months ago may be irrelevant. The system must track when data was collected and flag when it's stale. Stale data should be marked, not deleted — but it must be clearly distinguished from fresh data.

**Why**: Acting on stale intelligence is worse than having no intelligence because it creates false confidence. "We have data" feels like "we understand the market" — but if the data is 6 months old, you don't understand anything.

### Rule 7: If you're unsure whether an operation is extraction or interpretation, it's interpretation

When in doubt, draw the line toward the human. If you can't clearly say "this is just reporting a fact," then it's a judgment call and the human makes it.

**Why**: The bias should always be toward giving the human more decision authority, not less. You can always shift authority toward agents later as trust grows. You can never recover authority you gave away too early.

### STEP 2: WHO DOES WHAT
**⏱ Estimated: 45-60 minutes**

**What to do**: For each operation from Step 1, decide who performs it.

You have three actors:
- **Human** (Peter) — judgment, strategy, relationship decisions, anything with high consequences or requiring context
- **Agent** (AI agent) — extraction, measurement, structured assessment, flagging, filtering
- **System** (automated infrastructure) — scheduling, storage, alerting, data management, anything that runs without thinking

When deciding, ask yourself:
1. **Can this go wrong in a way that harms someone?** → Human
2. **Does this require understanding Peter's specific strategy and values?** → Human
3. **Is this a mechanical operation that doesn't require judgment?** → System
4. **Is this a structured analytical task with clear inputs and outputs?** → Agent

**Watch out for**:

### STEP 3: WHAT CAN GO WRONG
**⏱ Estimated: 30-45 minutes**

**What to do**: For each agent operation (not human, not system), identify what failure mode is most likely and what happens when it fails.

There are five characteristic failure modes for AI agents:

| Agent Type | Characteristic Failure Mode | What It Looks Like |
|---|---|---|
| **Extractor** | Hallucination | Reports something that isn't true. Confabulates details. Fills gaps with plausible-sounding fiction. |
| **Measurer** | Noise-as-signal | Treats random variation as meaningful trend. A 2% change becomes "significant movement." |
| **Assessor** | Overconfidence | States a flag or evaluation with high confidence when the evidence is weak. Fails to express uncertainty. |
| **Generator** | Fabrication | Creates content that includes made-up facts, fake citations, or unsupported claims. |
| **Aggregator** | Omission | Misses a critical signal because it didn't fit the pattern. Drops the outlier that mattered most. |

For each agent operation:
1. Identify the most likely failure mode
2. Describe what it would look like in practice (be specific to THIS workflow)
3. Rate the severity: 🟢 Embarrassing / 🟡 Misleading / 🔴 Dangerous
4. Describe what recovery looks like (how would you detect and fix it)

**Watch out for**:

- Noise-as-signal in market trend measurement. "Competitor X's Twitter mentions increased 15%" — is that a real signal or did someone with 100K followers mention them once? The measurer can't tell. The human must.

**Watch out for**:
- Hard constraints that are vague. "Be careful about accuracy" is not a hard constraint. "Never report a claim without citing a source URL" is a hard constraint.
