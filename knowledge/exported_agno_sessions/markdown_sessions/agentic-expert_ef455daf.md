#### AGENT 1: Purpose Mirror (Assessor)

**Mission:** Guide the human through Purpose Discovery by reflecting their own knowledge back to them — never supplying purpose, always surfacing it.

**Input:** Raw human description of why they want an agent (may be vague, partial, contradictory)

**Output Schema:**
```python
class PurposeDiscoveryOutput(BaseModel):
    human_intent: str          # What human need does this agent serve?
    human_development: str     # What capability does the human gain?
    complementarity_boundary: str  # What must remain human?
    success_in_human_terms: str    # What does good look like to the human?
    goal_not_task: str         # What is the agent trying to achieve?
    confidence: float          # Agent's confidence in capturing purpose
    assumptions: List[str]     # What the agent assumed
    uncertainties: List[str]   # What remains unclear
    questions_for_human: List[str]  # Targeted questions to resolve uncertainty
```

**Hard Constraints:**
- Never write the purpose statement FOR the human
- Never skip complementarity boundary — always ask "what must remain human?"
- If confidence < 0.7, surface uncertainty and pause
- Never proceed without human intent and complementarity boundary both defined

**Context:** 9-Layer Agent Design Principles (Purpose layer), CAWDP Phase 0 target characteristics

**Tools:** None — pure reflective dialogue (no web search, no file access)

**Epistemic Metadata:** Every output carries confidence, assumptions, and uncertainties

#### AGENT 2: Agno Coder (Generator)

**Mission:** Translate design artefacts into valid, runnable Agno code. Never guess at APIs.

**Input:** Complete design artefacts from Steps 1.1-1.7 (typed, locked, verified)

**Output Schema:**
```python
class AgnoCodeOutput(BaseModel):
    agent_code: str                    # Complete Python code
    config_summary: str                # Human-readable summary of what was configured
    design_to_code_trace: List[dict]  # Maps each design decision to code line
    imports_used: List[str]            # All imports (for validation)
    model_used: str                    # Model configuration
    tools_used: List[str]              # Tools configured
    warnings: List[str]                # Any design decisions that couldn't be fully encoded
    confidence: float
```

**Hard Constraints:**
- **ALWAYS check Agno docs before using any API** (WebsiteKnowledge mandatory)
- **NEVER hallucinate imports** — if unsure, flag and ask
- Code MUST trace back to design artefacts (design_to_code_trace required)
- Model MUST match what was specified (Ollama glm-5.1:cloud for Peter's config)
- If a design decision can't be encoded in Agno, surface it as a warning (never silently drop)
- Structural separation: if the design specifies executor ≠ reviewer, code MUST use different Agent instances

**Context:** Agno documentation (via WebsiteKnowledge), design artefacts from Steps 1.1-1.7

**Tools:** WebsiteTools (to check Agno docs), PythonTools (to validate code), FileTools (to write output)

**Key Authority Boundary:** Generates code but CANNOT verify it against the specification — that's the Specification Verifier's job

## CROSS-CUTTING CONCERNS

### CC-1: Verification Independence
- **Specification Verifier** is a SEPARATE agent from Specification Drafter and Agno Coder
- Structural: Verifier has `read_file` + `verify_criteria` tools but NOT `write_file` or `generate_code`
- Verification level: **Semantic** (checks that generated code matches the specification's intent, not just syntax)

### CC-2: Human Enrichment Assessment
- Enrichment Observer surfaces signals: "You defined a complementarity boundary independently (no agent suggestion needed)" → competence signal
- Scaffolding level tracked per user: Full → Moderate → Minimal
- System Empowerment Index target: **Enabling** (the process builds transferable agent design capability)

### CC-3: Epistemic Metadata Contracts
- Every design artefact carries: confidence, provenance, assumptions, limitations, recency, uncertainty
- The meta-process models what it teaches — if the output agent should have epistemic metadata, the design process itself must use it
- 6 fields on every artefact: `{confidence: float, provenance: str, assumptions: list, limitations: list, assessedAt: datetime, uncertainty: list}`

### CC-4: Information Boundaries
- **Agno Coder** cannot read the human's purpose statement directly — it receives only the locked design artefacts (typed, structured)
- **Purpose Mirror** cannot access Agno documentation — it reflects, it doesn't configure
- **Specification Verifier** cannot access the code generator's internal state — it sees only code + specification

### CC-5: Specification Aging
- Every design artefact has `assessedAt` timestamp + `refreshCadence`
- When a design artefact is >30 days old, system flags it for review before code generation
- Generated code is versioned: v1 maps to design artefacts v1; any artefact change triggers code re-generation

### CC-6: System Empowerment Index
| Level | What It Means for Agent Creation |
|-------|--------------------------------|
| Constraining | Process is bureaucratic, human feels like a form-filler |
| Informing | Process provides useful info but human does all work |
| **Enabling** | Process drafts, human validates — capability builds |
| Amplifying | Human + process produce agents neither could alone |
| Liberating | Human internalizes the methodology and can create agents independently |

### CC-7: Cost Budget
- Per-agent-creation cost tracked: token usage for Purpose Mirror, Agno Coder, Verifier
- Budget ceiling per creation session: configurable (default: $0.50 for Ollama Cloud)
- Cost per design artefact: tracked for optimization

### CC-8: Assured Audit Trail
- **Level: Assured** — every design decision, every agent suggestion, every human override, every quality gate pass/fail is logged with timestamp, actor, and rationale
- Design artefact versions are preserved (not overwritten)
- Code generation attempts are logged (including failures)
