## Agent Specifications

### **LAYER 1: Value Understanding**

#### **Personal Asset Cartographer**
```
Purpose: Builds and maintains a living inventory of your tangible and intangible assets

Inputs Required:
- Structured interviews about skills, experience, knowledge areas
- Work product samples (strategies, frameworks, designs)
- Network information (LinkedIn, professional contacts)
- Reflective prompts about passions and strengths

Outputs:
- Hierarchical asset map categorized by:
  • Hard skills (technical, domain expertise)
  • Soft skills (leadership, communication, design thinking)
  • Knowledge domains (AI, strategy, innovation, education)
  • Experience narratives (projects, roles, achievements)
  • Network assets (who you know, their domains, relationship strength)
  • Work product library (strategies, frameworks, methodologies)
  • Passion indicators (what energizes you)
  
- Confidence scores on each asset
- Asset maturity ratings (nascent, developing, mature, expert)
- Unusual asset combinations (your unique intersections)
```

#### **Capability Deep-Dive Agent**
```
Purpose: Goes beyond surface-level understanding to identify transferable and hidden capabilities

Behaviors:
- Asks probing questions: "You developed X skill in Y context—where else could that apply?"
- Identifies skill stacking opportunities
- Surfaces "underrated" capabilities you may not see as valuable
- Maps capability dependencies (what enables what)
- Detects capability gaps that, if filled, unlock new opportunities

Special Focus:
- Transfer value: "Your strategy frameworks developed for X could apply to Y with modification Z"
- Hidden assets: "You mentioned managing stakeholders in research contexts—this is valuable for..."
```

#### **Work Product Archaeologist**
```
Purpose: Excavates value from existing and past work products

Key Insight: Your strategies, opportunity assessments, innovation pipelines, and design thinking outputs ARE products

Behaviors:
- Catalogs all work products with metadata
- Identifies reusable components, patterns, and frameworks
- Scans for "productization potential" 
- Maps work products to potential customer segments
- Identifies incomplete work that could be finished and monetized

Output Examples:
- "Your opportunity assessment framework from Project X could be packaged as a consulting offering"
- "These strategy documents contain methodology patterns that could become a course module"
- "Your stakeholder analysis approach has consulting product potential"
```

#### **Network Value Mapper**
```
Purpose: Understands your professional network as a strategic asset

Behaviors:
- Maps network by domain, influence, relationship strength
- Identifies "super-connector" nodes in your network
- Scans for network gaps (what types of connections are missing)
- Tracks network health (stale relationships, dormant connections)
- Identifies who could be customers, collaborators, validators, mentors

Special Focus:
- "Who in your network is at companies facing problems you can solve?"
- "Who could provide warm introductions to target customers?"
- "Who has complementary skills for partnership opportunities?"
```

#### **Passion Signal Agent**
```
Purpose: Detects what energizes you vs. drains you

Rationale: Your best revenue opportunities align passion with market need

Behaviors:
- Analyzes patterns in what you discuss with enthusiasm
- Identifies energy patterns from past work experiences
- Flags opportunities that align with passions vs. those that drain
- Tracks passion drift over time

Output: Passion alignment score for each discovered opportunity
```

## Agent 3: Experiment Designer

**Purpose:** Convert opportunities into minimal, testable experiments—this is your anti-overthinking mechanism.

```

│ RESOURCES NEEDED:                                             │

│ Money: [Dollars]                                              │

│                                                                │
│ DURATION: [Time to get answer]                                │
│ Example: "2 weeks"                                            │
│                                                                │

│                                                                │
│ KILL CRITERIA: [What proves it's not worth it]               │
```

## Agent 4: Coordinator

**Purpose:** Manage the flow between agents, store results, and maintain your opportunity pipeline.

```
CORE FUNCTIONS:

1. SESSION MANAGEMENT
   - Initiates Asset Mapper when asset inventory is stale or missing
   - Routes to Opportunity Scout when asset map exists
   - Routes to Experiment Designer when opportunity is selected
   
2. PIPELINE TRACKING
   Maintains a simple opportunity pipeline:
   
   ┌─────────────────────────────────────────────────────────────┐
   │ OPPORTUNITY PIPELINE                                        │
   ├─────────────────────────────────────────────────────────────┤
   │                                                             │
   │ DISCOVERED │  SCORED  │  TESTING  │  ACTIVE  │  COMPLETED   │
   │            │         │           │          │              │
   │ [Opp 1]    │ [Opp 3] │ [Opp 5]   │ [Opp 7]  │ [Opp 9] ✓    │
   │ [Opp 2]    │ [Opp 4] │ [Opp 6]   │          │              │
   │            │         │           │          │              │
   └─────────────────────────────────────────────────────────────┘
   
   WIP LIMIT: Maximum 2 opportunities in TESTING at once
   
3. EXPERIMENT LOG
   Records results of each experiment:
   - Opportunity tested
   - Hypothesis
   - Test performed
   - Results
   - Decision (pursue/kill/pivot)
   - Key learnings

4. LEARNING CAPTURE
   - What patterns emerge from successful experiments?
   - What patterns emerge from killed opportunities?
   - What assets are being used most?
   - What customer segments show interest?

INTERACTIONS:

1. "What opportunities am I pursuing?"
   → Shows pipeline status

2. "I want to explore a new opportunity"
   → Routes to Opportunity Scout

3. "I'm ready to test an opportunity"
   → Routes to Experiment Designer

4. "My experiment is complete"
   → Captures results, updates pipeline

5. "What should I work on next?"
   → Reviews pipeline, recommends next action
```

## How the MVP Team Works Together

```
WORKFLOW:

1. INITIAL SETUP (One-time)
   Coordinator → Asset Mapper
   "Let's build your asset inventory..."
   [Structured conversation creates asset map]
   Coordinator stores asset map in memory

2. OPPORTUNITY DISCOVERY (On-demand)
   You: "What opportunities exist for me?"
   Coordinator → retrieves asset map → Opportunity Scout
   Opportunity Scout generates 5-10 opportunity candidates
   You select 1-2 to explore further

3. EXPERIMENT DESIGN (On-demand)
   You: "I want to test [opportunity name]"
   Coordinator → Experiment Designer
   Experiment Designer creates lean experiment plan
   You execute the test

4. RESULTS CAPTURE (After experiment)
   You: "Here's what happened..."
   Coordinator captures results and learnings
   Pipeline updated
   Recommendations for next action

5. ONGOING REFRESH (Periodic)
   Coordinator prompts: "Time to update your asset map?"
   Asset Mapper runs focused update
   New opportunities may emerge
```
