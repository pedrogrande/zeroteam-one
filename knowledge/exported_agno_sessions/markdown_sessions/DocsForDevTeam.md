Based on the repository structure and our session, here is a comprehensive list of the documentation types in the EdgeTrust / Trust Builder project:

1. 🧠 Agent Knowledge System
   The Spine: AGENT-HUB.md (The central navigation point for all agents).
   Quickrefs: project/trust-builder/quickrefs/_.md (Role-specific cheat sheets for Developer, QA, Advisor).
   Patterns: project/trust-builder/patterns/_.md (Reusable "Gold Standard" technical solutions).
   Meta-Docs: project/trust-builder/meta/\*.md (System blueprints, prompt changelogs, and the new Agent Performance Reports).
2. 📘 Strategic & Product Documentation
   Vision & Personas: 00-product-vision-and-goals.md, 01-user-personas-and-journeys.md.
   Requirements: 02-functional-requirements.md, 03-nonfunctional-requirements.md.
   Platform Specs: project/platform/\*.md (Platform-level constraints and ONE ontology definitions).
3. ⚙️ Technical Specifications
   Architecture: 04-data-model-and-api-design.md.
   Smart Contracts: 05-smart-contract-behaviour-spec.md.
   Migration: 08-migration-and-audit-strategy.md.
4. 🔄 Operational Artifacts (The "Paper Trail")
   Backlog: BACKLOG.md.
   User Stories: project/trust-builder/product-manager/stories/S#-##-story-name.md.
   QA Reports: project/trust-builder/product-manager/stories/S#-##-QA-REPORT.md.
   Strategic Reviews: project/trust-builder/product-manager/advisor-feedback/_.md (Pre- and Post-implementation reviews).
   Retrospectives: project/trust-builder/retros/sprint-X-retrospective.md and story-_-retro.md.
5. 🎓 Learnings & Guidelines
   Sprint Learnings: project/trust-builder/retros/sprint-X-learnings-and-guidelines.md (Codified policies derived from retros).
   Blueprints: AGENT-SYSTEM-BLUEPRINT.md (The guide for replicating this agent system).
6. 💻 Codebase Documentation
   Root: README.md (Setup and testing instructions), CLAUDE.md.
   Implementation Summaries: S4-01-IMPLEMENTATION-SUMMARY.md (Technical implementation notes).
   This structure implements the "3-Tier Knowledge Hierarchy" (Tier 1: Agent Specs, Tier 2: Quickrefs/Hub, Tier 3: Deep Specs) to optimize for agent token efficiency.
