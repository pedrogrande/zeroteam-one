# The Co-Design Process: Human + Agent Team

> **How to read this:** Each step is led by one **Process Agent**. The 🔵 **Blue Hat** is always present as the process steward. Other hats and personas are **summoned** — activated by the human when specific pressure is needed. The human's job at every step is to read the output, decide if it's good enough to proceed, and make the call.

***

## Phase 1 — Ground

### Step 1: Scan Real-World Examples

**Led by:** Scout Agent
**Scout's task:** Search for real organisations attempting something in your target space. Produce a structured brief: what exists, what's working, what feels cobbled together, and what's conspicuously absent.


| Active agents | Their role at this step |
| :-- | :-- |
| 🔵 Blue Hat | Ensures Scout stays descriptive — not evaluative yet |
| ⚪ White Hat | Strips assumptions from the brief: "Is this observed or inferred?" |
| 🔴 Red Hat | "What feels surprising or uncomfortable in what we found?" |

**Human's role:** Decide which real-world sectors to scan. Read the brief and mark what resonates as genuinely surprising versus what confirms existing assumptions.

**Output:** A 1-page brief: *Here is what exists. Here is what's missing. Here is what no one seems to have solved.*

***

### Step 2: Name the Framework Gaps

**Led by:** Gap Finder Agent
**Gap Finder's task:** Take the Scout brief and convert observations into named, specific open questions — the gaps the thought experiment will deliberately probe.


| Active agents | Their role at this step |
| :-- | :-- |
| 🔵 Blue Hat | Checks that questions are genuinely open, not leading |
| ⚫ Black Hat | "Which of these gaps might not actually be gaps — just things we haven't found yet?" |
| 🟢 Green Hat | "What gaps might exist that we haven't even thought to look for?" |
| **Sceptic Persona** | "Why hasn't anyone solved this? Is it actually solvable or is there a reason?" |

**Human's role:** Read the gap list and *prioritise*. Choose the 2–3 gaps that feel most important to probe. This is the irreversible call — it sets the direction for everything that follows.

**Output:** A named list of 2–3 open questions. These become the *design brief* for the thought experiment. Every case in Phase 2 will be chosen to stress-test one of these questions.

***

## Phase 2 — Imagine

### Step 3: Set the Future Frame

**Led by:** World Builder Agent
**World Builder's task:** Write a scenario brief for a specific future horizon. Define: what has changed by then, what hasn't changed, and — critically — write an explicit list of hypotheses, not predictions.


| Active agents | Their role at this step |
| :-- | :-- |
| 🔵 Blue Hat | Enforces the rule: *outputs are hypotheses to test, not conclusions to reach* |
| ⚪ White Hat | "What do we actually know about this future versus what we're projecting?" |
| 🟡 Yellow Hat | "What's the most genuinely optimistic version of this world that's still plausible?" |
| 🔴 Red Hat | "What about this future feels exciting? What feels unsettling?" |

**Human's role:** Read the scenario and confirm it feels like a plausible probe of the gaps named in Step 2 — not just an interesting world to explore. Adjust the horizon if needed.

**Output:** A scenario brief with: the future year, 3–5 changed conditions, 1–2 unchanged constraints, and an explicit list of hypotheses the cases will test.

***

### Step 4: Design the Fictional Use Cases

**Led by:** Case Architect Agent
**Case Architect's task:** Propose 2–3 fictional organisations *specifically chosen* to stress-test the named gaps. Each case should occupy a different sector, scale, or context — not three similar organisations dressed differently.


| Active agents | Their role at this step |
| :-- | :-- |
| 🔵 Blue Hat | Checks each case is genuinely testing a different gap — no overlap |
| 🟢 Green Hat | "What's an unexpected sector that would reveal something the obvious choices wouldn't?" |
| ⚫ Black Hat | "Is any of these cases too convenient — designed to confirm rather than challenge?" |
| **Regulator Persona** | "Which of these organisations would face the most complex compliance surface? Is that gap in the brief?" |

**Human's role:** Choose which 2–3 cases to proceed with. Reject any case that feels like a good illustration rather than a genuine probe. The cases should make you slightly uncomfortable — that discomfort is the signal they're doing real work.

**Output:** 2–3 case briefs: sector, founding context, what they do, and which gap from Step 2 they are specifically stress-testing.

***

### Step 5: Deep Dive Each Case

**Led by:** Deep Diver Agent
**Deep Diver's task:** For each case, produce a full internal analysis across: agent operations, team structure, costs and revenue, what humans actually do, how the team grows, and — most importantly — *challenges*. The challenges section is non-negotiable. If it's short, the dive isn't deep enough.


| Active agents | Their role at this step |
| :-- | :-- |
| 🔵 Blue Hat | Ensures challenges section is genuinely critical, not a polite afterthought |
| ⚫ Black Hat | Operates continuously during this step: "What else could go wrong here?" |
| 🟡 Yellow Hat | "What's the strongest case for why this works — what are we underselling?" |
| **End User Persona** | "How does the actual person being served experience this? What would frustrate or confuse them?" |
| **Operator Persona** | "What would it actually feel like to run this day-to-day? What would break you?" |
| **Future Worker Persona** | "How has my role changed? Am I more capable or more managed?" |

**Human's role:** Read each dive and mark every challenge that feels like it *maps to a real problem you've already seen* — these are the most valuable signals. Also mark anything that feels too neat or unearned.

**Output:** 3 full case documents. The challenges sections are the harvest-ready material for the next step.

***

### Step 6: Extract Cross-Case Patterns and Gaps

**Led by:** Harvester Agent
**Harvester's task:** This is the most important step in the process. Read all three case documents and produce a standalone gap report: *what did all three cases reveal that the original framework didn't anticipate?* This is not a summary — it is new knowledge.


| Active agents | Their role at this step |
| :-- | :-- |
| 🔵 Blue Hat | Enforces a clean separation: "Is this a pattern or a coincidence?" |
| ⚪ White Hat | "Which patterns are genuinely observed across cases versus projected onto them?" |
| 🟢 Green Hat | "What unexpected pattern showed up that no one put in the brief?" |
| 🟡 Yellow Hat | "What's the most exciting insight in here — the one that changes how you'd design something?" |
| **Sceptic Persona** | "Which of these 'insights' would a practitioner dismiss immediately? Why?" |

**Human's role:** This is the highest-value human act in the entire process. Read the gap report and mark: (a) which gaps are *strong signals* — things you'd bet on being real, and (b) which are *interesting hunches* — worth noting but not designing around yet. This distinction belongs to you, not the agents.

**Output:** A gap report in two columns — *Strong Signals* and *Interesting Hunches*. This document is the bridge between the imagined future and real-world action.

***

## Phase 3 — Focus

### Step 7: Define the Real Person

**Led by:** Persona Maker Agent
**Persona Maker's task:** Build a persona and empathy map for the real human who would interact with whatever you build. Crucially, this persona is built *after* the thought experiment, so the gaps discovered in Step 6 sharpen what you look for — not the fictional world you created.


| Active agents | Their role at this step |
| :-- | :-- |
| 🔵 Blue Hat | Checks that the persona is shaped by the gap report, not by the fictional cases |
| 🔴 Red Hat | "What does this person *feel* about the problem — not just what do they think or do?" |
| ⚪ White Hat | "Which parts of this persona are assumptions? What would we need to validate with real people?" |
| **End User Persona** | Cross-checks the built persona against their own lived experience |

**Human's role:** Mark every assumption in the persona that needs real-world validation. The persona is a *starting hypothesis*, not a conclusion.

**Output:** A persona and empathy map with assumptions explicitly flagged for validation.

***

### Step 8: Separate Signals from Hunches

**Led by:** Question Keeper Agent
**Question Keeper's task:** Consolidate everything surfaced across the process into a clean, honest list of open questions. Separate *strong signals* (multiple cases, backed by pattern, connected to a real person's experience) from *interesting hunches* (single observation, not yet connected to real data).


| Active agents | Their role at this step |
| :-- | :-- |
| 🔵 Blue Hat | Final process check: "Are we moving toward action prematurely?" |
| ⚫ Black Hat | "Which strong signals might be wrong — what's the counter-argument?" |
| 🟢 Green Hat | "What question should be on this list that we haven't thought to ask?" |
| **Investor/Funder Persona** | "Which of these questions, if answered, would most change whether I'd back this?" |
| **Regulator Persona** | "Which of these questions, if left unanswered, creates the most risk?" |

**Human's role:** Read the list and confirm it feels honest — not over-confident and not under-confident. This list becomes the brief for everything built next.

**Output:** Two lists — *Strong Signals to Act On* and *Interesting Hunches to Watch*.

***

## Phase 4 — Sketch

### Step 9: Generate Experiments, Not Products

**Led by:** Concept Sketcher Agent
**Concept Sketcher's task:** For each strong signal, sketch one idea as a *low-cost experiment to run*, not a product to build. Each sketch should answer: what's the smallest version of this we could test with real people in 4–6 weeks?


| Active agents | Their role at this step |
| :-- | :-- |
| 🔵 Blue Hat | Enforces: "Is this an experiment or is it already a product? If it can't be tested cheaply, resize it." |
| 🟡 Yellow Hat | "What's genuinely exciting here — what's the version of this that could really matter?" |
| ⚫ Black Hat | "What's the most likely reason this experiment produces no useful signal?" |
| 🟢 Green Hat | "What's the version of this no one has thought to try yet?" |
| **Operator Persona** | "Could I actually run this experiment? What would make it impossible in practice?" |
| **End User Persona** | "Would I participate in this? What would make me say no?" |

**Human's role:** Choose which experiments to run. Frame each one as: *We believe [signal]. We'll test this by [action]. We'll know it's working when [observable outcome].* This framing is yours — no agent should write it for you.

**Output:** 2–3 experiment briefs, each with a clear hypothesis, a minimum viable test, and a success signal.

***

## The Human's Non-Negotiable Roles

Across all 9 steps, these acts belong to you — they cannot be delegated to any agent:

- **Choosing which gaps matter** (Step 2) — this is your values and judgment, not pattern matching
- **Selecting which cases to build** (Step 4) — cases should make you uncomfortable, and only you can feel that
- **Distinguishing strong signals from hunches** (Steps 6 and 8) — this requires lived experience no agent holds
- **Writing the experiment hypotheses** (Step 9) — the framing of what you believe and what you're testing is a commitment that must be yours
- **Deciding when to loop back** — if Step 6 reveals that Step 2 named the wrong gaps, the 🔵 Blue Hat will flag it, but you decide whether to go back[^1]
