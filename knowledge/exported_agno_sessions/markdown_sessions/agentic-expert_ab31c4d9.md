# Phase 1 — Task Intelligence

## Subtask Decomposition

Five subtasks. Start to finish for a single LinkedIn post.

---

### ST1 — Idea Capture & Typing

**What happens:** Peter has an idea. It gets captured with near-zero friction and classified.

| Element | Detail |
|---------|--------|
| **Input** | Raw idea fragment (text, voice, bullet points) |
| **Process** | Capture the idea. Classify by post type. Tag strategic purpose. |
| **Output** | A **typed idea record**: type (Insight / Contrarian / Evidence / Behind-the-Scenes / Challenge), purpose (awareness / credibility / engagement / conversion), the raw idea text |
| **Cognitive operations** | **Mechanical** (capture) + **Analytical** (classification) |

---

### ST2 — Research & Evidence Grounding

**What happens:** The typed idea gets grounded with evidence — data, examples, references. This is where "epistemic honesty" lives: *I believe this because…*

| Element | Detail |
|---------|--------|
| **Input** | Typed idea record from ST1 |
| **Process** | Search for supporting evidence. Find counterpoints (especially for Contrarian type). Identify the *because* behind the claim. |
| **Output** | An **evidence pack**: supporting data points, counterpoints, source references, a "grounds" statement (what this idea rests on), confidence level |
| **Cognitive operations** | **Analytical** (research, search, compare) + **Generative** (synthesis — combining sources into a coherent grounding) |

---

### ST3 — Draft Generation

**What happens:** The grounded idea becomes a LinkedIn post. Draft style adapts to post type.

| Element | Detail |
|---------|--------|
| **Input** | Typed idea record + evidence pack |
| **Process** | Write the post. Adapt structure and tone to post type (a Contrarian hook ≠ an Evidence hook). Apply Peter's voice profile (Futuristic, Strategic, Ideation, Activator energy — not generic AI). |
| **Output** | A **typed draft**: the post text, metadata (type, purpose, hook style), word count, estimated engagement type |
| **Cognitive operations** | **Generative** (writing) + **Evaluative** (voice matching — does this sound like Peter?) |

---

### ST4 — Review & Brand Alignment

**What happens:** The draft gets checked against four dimensions before Peter ever sees it.

| Element | Detail |
|---------|--------|
| **Input** | Typed draft from ST3 |
| **Process** | Check factual accuracy against evidence pack. Check brand alignment (traces to "Principles aren't production"?). Check voice (Futuristic/Strategic/Ideation/Activator — not bland). Check epistemic honesty (claims grounded, or flagged as belief/opinion?). Flag risks. |
| **Output** | A **review report**: pass/fail per dimension, flagged issues, suggested fixes, confidence score per dimension, risk flags (tone, factuality, brand drift) |
| **Cognitive operations** | **Evaluative** (assessment) + **Analytical** (comparison against standards) |

---

### ST5 — Publish Decision

**What happens:** Peter sees the reviewed draft and decides.

| Element | Detail |
|---------|--------|
| **Input** | Reviewed draft + review report |
| **Process** | Peter reads. Approve → publish. Revise → back to ST3. Reject → capture why, feed back to ST1. |
| **Output** | A **publish command** (approve/revise/reject) + Peter's feedback |
| **Cognitive operations** | **Evaluative** (human judgment) — **HUMAN ONLY** |

---

## Cognitive Operation Summary

| Subtask | Mechanical | Analytical | Generative | Evaluative | Intuitive |
|---------|:----------:|:----------:|:----------:|:----------:|:---------:|
| ST1 Capture & Typing | ● | ● | | | |
| ST2 Research & Grounding | | ● | ● | | |
| ST3 Draft Generation | | | ● | ● | |
| ST4 Review & Alignment | | ● | | ● | |
| ST5 Publish Decision | | | | ● | ● |

**Pattern:** The pipeline moves from extraction/capture → analytical grounding → generative drafting → evaluative review → human evaluative judgment. Each stage shifts the dominant cognitive operation. This is clean — no subtask bundles wildly different operations.

---

## Edge Case Inventory

| # | Edge Case | Why It Matters | Which Subtask |
|---|-----------|---------------|---------------|
| E1 | **Idea has no evidence** — great insight, nothing to ground it with | Contrarian and Behind-the-Scenes types may legitimately lack external evidence. Pipeline must handle this without blocking. | ST2 |
| E2 | **Generic AI voice** — draft sounds like every other LinkedIn post | Brand death. The #1 quality risk. | ST3 |
| E3 | **Factual error in draft** — agent fabricates a stat or misattributes | Trust destruction. Epistemic honesty principle violated. | ST4 |
| E4 | **Brand contradiction** — post contradicts a previous post or core position | Credibility erosion over time. | ST4 |
| E5 | **Frustration loop** — Peter rejects multiple drafts | RSD trigger. System must make rejection low-cost and feed back cleanly. | ST5 → ST3 |
| E6 | **Stale idea** — captured weeks ago, may no longer be relevant | Specification aging applied to ideas. | ST1 |
| E7 | **Multiple ideas compete for one slot** | Ideation strength means idea surplus, not shortage. Need prioritisation, not just FIFO. | ST1 |
| E8 | **Sensitive topic** — tone misfire could provoke pile-on | Low-reversibility territory. Extra caution at review. | ST4 |

---

That's Phase 1. Five subtasks, eight edge cases, clean cognitive operation profile.

# Phase 2 — Capability Allocation

## Reversibility Classification

Every subtask gets a reversibility rating. This is the **governor** — it determines how much autonomy any actor gets.

| Subtask | Reversibility | Rationale |
|---------|:------------:|-----------|
| ST1 Capture & Typing | **High** | Nothing published. Discard freely. |
| ST2 Research & Grounding | **High** | Nothing published. Wrong evidence gets corrected at ST4. |
| ST3 Draft Generation | **High** | Nothing published. Draft can be rewritten or binned. |
| ST4 Review & Alignment | **High** | Nothing published. Review flags are advisory, not final. |
| ST5 Publish Decision | **Medium** | LinkedIn posts *can* be deleted — but screenshots exist, and reputational impact lingers. Not fully reversible. |

**Governor insight:** ST1–ST4 are high-reversibility territory — agents can operate with more autonomy here because mistakes cost nothing. ST5 is the one place where the cost of error jumps. This is where human authority is non-negotiable.

---

## Ternary Complementarity Matrix

Scoring on a 1–10 scale across capability dimensions relevant to this pipeline. Scores reflect **Peter specifically** — his strengths profile, ADHD profile, and brand context — not a generic human.

### ST1 — Idea Capture & Typing

| Dimension | Human (Peter) | Agent | System | Gap (H–A) | Allocation |
|-----------|:---:|:---:|:---:|:---:|-----------|
| Capture speed & zero-friction | 3 | 8 | 9 | — | **System** |
| Idea quality (is this worth posting?) | 9 | 4 | 2 | 5 | **Human** |
| Type classification | 7 | 8 | 3 | — | **Agent** |
| Strategic purpose tagging | 8 | 6 | 3 | 2 | **Human** |

**Allocation: Collaborative (System→Agent→Human)**
- System provides the capture surface (zero-friction input — respects ADHD principle #8)
- Agent classifies type and drafts purpose tags
- Human originates the idea and confirms the strategic purpose
- **No 6+ gap → no human-only subtask here**

---

### ST2 — Research & Evidence Grounding

| Dimension | Human | Agent | System | Gap (H–A) | Allocation |
|-----------|:---:|:---:|:---:|:---:|-----------|

| Evidence relevance (is this evidence *worth* using?) | 8 | 6 | 2 | 2 | **Human** |
| Counterpoint identification | 6 | 8 | 2 | — | **Agent** |
| Confidence assessment | 8 | 5 | 2 | 3 | **Human** |

**Allocation: Agent-primary with human check**
- Agent does the heavy lifting: search, retrieve, find counterpoints, assemble evidence pack
- Human assesses relevance and confidence — because Strategic strength means Peter sees *which evidence matters* faster than any agent
- **No 6+ gap → no human-only subtask here**

**Allocation: Collaborative (Agent generates, Human voice-checks)**
- Agent drafts the structure and prose (volume & fluency)
- Human judges voice authenticity — *does this sound like me?*
- Gap of 4 on voice authenticity: not human-only, but human MUST review this dimension
- **Key insight:** the draft is the *highest-risk point for generic AI voice* (edge case E2). Human voice check is structural, not optional.

### ST4 — Review & Brand Alignment

| Dimension | Human | Agent | System | Gap (H–A) | Allocation |
|-----------|:---:|:---:|:---:|:---:|-----------|
| Factual verification | 6 | 8 | 5 | — | **Agent** |
| Brand alignment judgment | 9 | 5 | 3 | 4 | **Human** |
| Voice authenticity assessment | 9 | 5 | 2 | 4 | **Human** |
| Risk sensing (tone/controversy) | 8 | 4 | 2 | 4 | **Human** |
| Epistemic honesty check | 7 | 7 | 2 | 0 | **Shared** |

**Allocation: Collaborative (Agent checks facts, Human checks everything else)**
- Agent: factual cross-checking against evidence pack (structured, mechanical)
- Human: brand alignment, voice, risk sensing — all require Peter's lived understanding of what Future's Edge stands for
- Gap of 4 across three human-superior dimensions: not 6+, but these are **mandatory human review dimensions**
- **Conditional 6+ gap on sensitive topics (E8):** when a topic touches controversy, reputational risk, or potential pile-on → **risk sensing jumps to H=9/A=3, gap=6 → human-only for risk assessment on those posts**

| Dimension | Human | Agent | System | Gap (H–A) | Allocation |
|-----------|:---:|:---:|:---:|:---:|-----------|
| Final judgment (publish/don't) | 10 | 3 | 2 | **7** | **Human ONLY** |
| Reputational risk assessment | 9 | 4 | 2 | 5 | **Human** |
| Timing judgment (when to post) | 7 | 5 | 4 | 2 | **Human-led** |

**Allocation: HUMAN ONLY**
- 7-point gap on final judgment — the clearest human-only subtask in the pipeline
- No agent decides what gets published under Peter's name. Ever.
- System *can* suggest timing based on engagement data — but Peter decides

---

## Ternary Allocation Summary

| Subtask | System | Agent | Human | Mode | 6+ Gap? |
|---------|:------:|:-----:|:-----:|------|:--------:|
| ST1 Capture & Typing | Capture surface | Classify & tag | Originate & confirm | **Collaborative** | No |
| ST2 Research & Grounding | — | Search, retrieve, counterpoints | Relevance & confidence | **Agent-primary** | No |
| ST3 Draft Generation | — | Draft structure & prose | Voice check | **Collaborative** | No |
| ST4 Review & Alignment | — | Fact-check | Brand, voice, risk | **Collaborative** | Conditional* |
| ST5 Publish Decision | — | — | Approve / Revise / Reject | **Human ONLY** | Yes (7) |

*\*ST4 risk sensing hits 6+ gap when topic is sensitive/controversial — risk assessment becomes human-only for those posts.*

---

## One Key Design Decision

The 4-point gaps on **voice authenticity** (ST3) and **brand alignment** (ST4) don't reach the 6+ human-only threshold — but they're **mandatory human review dimensions**, not optional. The difference:

- **Human-only** = agent doesn't do this at all
- **Mandatory human review** = agent can do preliminary work, but human sign-off is structural (not skippable)

This distinction matters for how we architect ST3 and ST4. The agent drafts and reviews — but Peter's check on voice and brand is a **quality gate**, not a nice-to-have.
